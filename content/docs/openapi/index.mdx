## Quickstart

### Authentication Overview

This API uses a **OAuth-based authentication** mechanism to ensure secure access. Developers can generate an Okto Auth Token using the provided Okto Dev Tools and must include this token in the Authorization header for all protected endpoints.

### Authentication Flow:
- Generate Okto Auth Token:
Use the [Okto Dev Tools]('/tools') to generate an authentication token. This tool simplifies the token generation process.

- Use Access Token:
Include the token in the Authorization header for all API requests:
```bash
Authorization: Bearer <okto_auth_token>
```

- Session Expiry & Refresh:
A session is created for the user and client, with a default validity of 7 days. However, this duration can be customized as needed
